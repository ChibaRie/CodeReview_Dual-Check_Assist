# 系统更新日志

本目录记录代码评审系统自身的功能更新。

## 日志格式

每条记录包含：

- **更新时间**：YYYY-MM-DD HH:MM
- **更新类型**：feat / fix / refactor / docs / chore
- **更新摘要**：一句话描述
- **涉及文件**：变更的文件列表
- **使用方式**：如何使用新功能
- **验证结果**：测试或验证结论
- **后续注意**：需要关注的遗留事项

## 命名规范

日志文件按月份命名：`YYYY-MM.md`（如 `2026-07.md`）。

更新日志与日常操作日志（`system/logs/`）分开 — 只有涉及系统功能、规则、脚本、索引或目录结构的变更才写入这里。
